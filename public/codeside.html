Diagnosis — AuthFlow SaaS · Security & Backend Review

Reviewed by Dan K. · 2.5 days turnaround

3 findings
CRITICAL

JWT secret hardcoded in source — exposed in repo history

Rotate this immediately, even before fixing the code. Anyone with repo access can forge authentication tokens for any user.

src/lib/auth.ts:14 — move to environment variable, rotate immediately
HIGH

No row-level authorisation on /api/users/:id — IDOR

Any logged-in user can read any other user's data by iterating numeric IDs. Validates auth but not ownership.

src/api/users/[id].ts:38 — add: if (session.userId !== params.id) return 403
MEDIUM

No rate limiting on auth endpoints — brute-force is wide open

POST /api/login and /api/auth/reset have no throttling. A 6-digit OTP brute-force would succeed in under a minute.

consider: upstash/ratelimit or express-rate-limit on auth routes
DK

"Two ship-blockers. Fix the hardcoded secret and the IDOR today — they're not optional. The rate limiting matters but can follow in the next sprint. This app has good bones; just patch the holes."

— Dan Kowalski, Application Security · 13 yrs
